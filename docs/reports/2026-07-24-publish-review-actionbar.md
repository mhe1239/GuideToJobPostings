# 관리자 초안 검수 액션바 개선

## 날짜

2026-07-24

## 작업 목표

- 공고 생성 화면의 검수 동선을 단순하게 정리합니다.
- 체크박스 완료 여부로 승인을 제한하지 않고, 관리자가 마지막 확인 사항을 인지하게 합니다.

## 변경 사항

- 공고 생성 화면의 별도 `관리자 검수 체크` 카드를 삭제했습니다.
- 생성 초안 입력 항목 하단에는 별도 검수 버튼을 두지 않고, 하단 고정 액션바에서 검수 안내와 공개 결정을 함께 처리하도록 단순화했습니다.
- 하단 고정 액션바는 초안을 생성한 뒤에만 나타나도록 변경했습니다.
- 액션바의 `초안 생성`과 `수정` 버튼을 제거하고 `보류`, `공개 승인`만 유지했습니다.
- 액션바에 `관리자 검수 체크`와 `마지막으로 확인하셨나요?` 안내를 표시합니다.
- 검수 체크박스와 승인 잠금 기능은 제거하고, 제목·일정·지원 자격·답변 근거 확인을 안내하는 인지 기능만 유지했습니다.
- `보류` 처리 후에는 생성 초안과 선택 상태를 미생성 화면으로 초기화하고 하단 액션바를 숨깁니다.
- 하단 액션바 오른쪽 위에 닫기 버튼을 추가해 초안을 유지한 채 액션바만 숨길 수 있게 했습니다.
- `학교 홈페이지에서 가져오기` 입력 방식을 벗어나면 해당 초안과 액션바가 자동으로 사라지도록 했습니다.
- 공고 생성 화면 하단의 상태 목록을 `보류 목록`으로 변경하고 보류된 공고만 표시합니다.
- 공개 공고 수정·삭제 화면에는 공개 승인된 공고만 표시하도록 상태별 관리 화면을 분리했습니다.
- 초안 생성과 처리 완료 안내는 화면 중앙에 표시하며, 보류 완료 안내는 위험 상태 색상으로 구분합니다.
- 상세 공고 페이지의 시험용 녹색 팔레트를 제거하고 기존 파란색으로 되돌렸습니다.
- 웹과 Tauri 앱에 동일한 검수 흐름을 반영했습니다.

## 검증

- 웹 `npm test`: 19개 테스트 통과
- 웹 `npm run lint`: 통과
- Tauri `npm run test:preview`: 통과
- Tauri `npm run build`: 통과 및 macOS DMG 생성 확인
- 초안 생성 전 액션바 숨김, 생성 후 노출 확인
- 검수 체크박스 없음, 액션바 버튼 두 개만 존재하는지 확인
- 생성 초안 하단의 중복 검수 버튼이 제거되었는지 확인
- 보류 후 초안 미생성 화면 복귀, 액션바 숨김, 빨간색 완료 안내 확인
- 닫기 버튼으로 액션바만 숨기고 초안 내용은 유지되는지 확인
- 학교 홈페이지 가져오기 영역 이탈 시 액션바와 이전 초안이 자동으로 숨겨지는지 확인
- 공고 생성 화면에는 보류 공고만, 공개 공고 관리 화면에는 공개 공고만 표시되는지 확인
- 모바일에서 하단 고정 바가 내용을 가리지 않도록 여백과 2열 버튼 배치 확인

- 실제 관리자 계정으로 로그인한 브라우저 최종 화면 확인이 필요합니다.
